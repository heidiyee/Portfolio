Clutch

based on design from Turnstyle, built using:

  • html
  • sass
  • jQuery
  • gulp
  • neat

Clutch was a really fun project to work on and I think it’s my best website I’ve made so far. The website was very well designed and each page came with a mobile and desktop layout. This gave me a good starting point so all I had to do was focus on the code. There were a couple challenges along the way, including working with more complicated CSS3 transitions and learning how to test for browser support. In all, everything went well with this website and I’m very happy with the result.

Lessons Learned

  • grid system
  • ie
  • virtualbox
  • polyfill/shims

Before I started on the Clutch project, I wanted to make sure I understood what I could have done with the design layout of the Rock Box site. I was still a bit frustrated with the alignment issues and decided to ask my boyfriend about it, who is also a front-end developer. I try not to ask him too many questions, but this was one I couldn’t shake. He told me there was a thing called grid systems and I should look up one to use for the site. I spent some time googling and came across a framework called Neat by ThoughtBot. I decided to give it a shot since it was well documented and easy to incorporate into my gulp file.

Thank goodness for the grid system. Neat was very easy to understand and saved me so much time positioning elements. I used to use calc() for everything, but now, I just use the grid system. I’m pretty sure I’m not using it to its full capabilities and I still don’t know if Neat is the best framework out there since I still need to include bourbon in my projects, which I don’t use at all. However, the grid system really changed how the site turned out and I’m really happy I ended up using it on this project.

The next lesson learned I’m not exactly excited to write about. The lesson learned was about browser testing, specifically browser testing for Internet Explorer.

Omg, wtf.

Of course, I knew this day would come. I knew I would need to test all the projects I had done in different browsers besides Chrome before making them public to the world, but I kept putting it off. Luckily for me, front end developers have had to deal with this for years, so I didn’t have trouble finding out how to run IE on my mac. I use VirtualBox since it’s free and downloaded IE8 - IE11 from the microsoft site. Oh goodie, now I can use IE if I wanted to…

Anyway, the sites came out looking terrible, but I quickly learned about polyfills and shims. I used a polyfill called placeholder.js if anyone is interested. It works really well for placeholders in forms. Once I was done with the Clutch site, I made sure to check all the other projects I had done. I also made sure to test in Firefox and Safari, which weren’t bad at all.
