Rock Box

Based on design from Turnstyle

  • HTML5
  • SASS
  • jQuery
  • Gulp
Check it out

Rock Box is a local karaoke place here in Seattle, so I was very excited to start working on this project. Here is a link if you want to check out their actual site - www.rockboxseattle.com, which could really use a makeover. This design also came from Turnstyle - cause they’re awesome.

Since this was my third project and I was getting really comfortable with HTML, CSS, and jQuery, I wanted to try out SASS. I had already heard a lot about SASS from other developers and was told it could really help organize your project. With SASS, I also learned Gulp and am so happy I did. Gulp is pretty easy to use and gave me the ability to auto reload my page, which I loveeee, as well as consolidate everything into one final folder. Awesome stuff.

Lessons Learned

  • Web Design
  • Illustrator

Rock Box turned out to be a really difficult project to work on for a couple reasons, mostly because I didn't have a designer to collaborate with. The design came in a desktop layout, so I had to create it’s mobile layout, which was a challenge for me since I didn’t have to for the last two projects. One of the challenges was incorporating all the images on a small screen. I decided to take out the bigger images, such as the person singing, until it was viewed on a wider screen. I think this helped a lot and I feel like the mobile layout turned out well.

Another concept I learned was the hero doesn’t have to be the full height of the screen. I had used 100vh for the hero section in Fulcrum, so I thought that would be the same case for any hero image on any site. This is not true and I found out this can give you issues if the height of the screen is not large enough to fit all the content. I started to notice that text would scrunch together in smaller mobile screens. I ended up making a height media query to fix this, but this taught me a valuable lesson that every site is different and what I did on the last site might not work well on the next.

As I started to build out the desktop layout of Rock Box, I noticed the sections weren’t aligning to each other and the site didn’t look cohesive. Since I was using SASS to build the project, each section had its own SASS file. This came with its pros and cons. The pros were it kept my project organized and I was able to build each section’s CSS independent from each other. However, that also brought a con with it, which was that the margins and paddings of each section may not align with each other, especially because the content switched between text-align center or text-align to the left. If you looked at the project from top to bottom, you didn’t see any vertical alignment. For example, from the picture below, you can see that the footer doesn’t align with the section above it. I struggled a lot with this and ended up leaving it the way it was because I didn’t know how to fix it. The solution wasn’t found until my next project, Clutch.

Lastly, I learned a lot more about Illustrator. Since the Fulcrum site already came with a design for mobile and desktop, I didn’t have to do much except save images. With Rock Box, I learned more about the differences between the types of image files you can use - svg vs jpg vs gifs and saving for web. I also learned how to use an artboard and how to save the images piece by piece using an artboard so the canvas size is still the same. I really like using illustrator now and even used it to create the design layout for my portfolio.
